GLIA

Gnu · LInux · Ai

A Linux distribution with a built-in AI terminal assistant.

Not a chatbot that uses your PC for you — a bridge that brings you to the terminal. Every request in plain language becomes a real shell command, shown to you before it runs: you read it, you approve it, you learn it. Little by little the terminal becomes your own personal textual desktop.

Development status — Debian & Fedora are work in progress

The assistant is built to run on Arch, Debian and Fedora, but so far it has been fully tested only on Arch / CachyOS. Debian and Fedora support is under active development and not yet tested. Testers and contributors are very welcome — open an issue or a pull request with your results.

Why now

Built for the near future

A wager on the next few years — and on who the terminal is really for. As local AI becomes ordinary, the question isn't whether your computer will have an assistant, but whether it hands you the machine or takes it away.

GLIA · manifesto

AI models keep getting smaller and sharper; personal machines keep getting more powerful. Soon, a genuinely capable assistant running entirely on your own computer will be ordinary, not exotic.

That's a fork. All that local power can make the computer a black box you talk at and stop understanding — or the best teacher the command line has ever had.

GLIA takes the second road: it doesn't use the machine for you, it hands it to you — from zero if you're starting out, with far more reach if you already know your way around. The shell newcomers of that near future deserve to be helped in, not handed a black box.

Positioning

More than an English → shell translator

Plenty of tools now turn plain language into a shell command — a crowded, useful idea. GLIA's difference isn't the translation. A plain translator runs one way: words in, a command out that runs. GLIA closes the loop — every command is handed back to you, to read and to learn, so a request returns to you as knowledge.

It teaches, not replaces

Every proposal also shows the real command, so you learn the terminal instead of outsourcing it. The AI steps aside the moment you know what you want.

Green by design

Runs on local Ollama — no cloud, no API keys, nothing leaves your machine. Repeat a task and it becomes a shortcut that runs with no AI call; a cache serves repeated questions. Fewer model calls means less energy and a faster terminal.

A whole distribution

Not just a command: GLIA ships as a minimal Linux distro (live ISO + Calamares installer) that boots straight to the terminal with the assistant onboard — and the same glia installs onto the Arch, Debian or Fedora you already run.

The translation is the easy part — and GLIA gets it right. What makes it GLIA is everything built on a tool that actually works: it teaches while it runs, it's green by design, and it's a whole system, not a lone command.

Philosophy

Support, not substitution

In the brain, glia cells support, feed and protect the neurons — without ever thinking in their place. GLIA does exactly that at the terminal: it never takes it over, and it's not a chatbot that writes the commands for you and pulls you away. It stays beside you, feeds your curiosity, and helps you discover and truly master the Linux command line yourself.

Transparency

You always see the command, never just the result. The AI proposes, you decide.

Learning by doing

Each interaction is a small terminal lesson. What you learn here transfers everywhere.

Safety first

Dangerous commands like rm -rf, dd, mkfs require reinforced confirmation.

Fully local

Runs on Ollama + aichat on your own machine. No cloud, no API keys, offline.

Standard conventions

Flags mirror real tools: -h, -V, -a. Never bespoke syntax when a standard exists.

Everything logged

Every executed command is saved to ~/.local/share/glia/glia.log for review.

No lock-in — build your own

GLIA boots to a minimal terminal with the assistant already onboard. You add a desktop environment, a browser or a dev stack yourself, by asking glia — never bound to a distribution's defaults from day one, and just as free to bring the assistant to the Arch, Debian or Fedora you already prefer.

Make it yours

Rename the assistant to whatever you like — glia --rename mypc — and it answers to that name. The glia command always stays active as a fallback, so you can never lock yourself out of your own assistant.

Get started

Install GLIA — three ways

There are no prebuilt ISO downloads: the ISO is always built fresh from the repo. The quickest way to try GLIA is to add the assistant to the Linux you already run.

Why there's no ready-made ISO

Because the machine should be yours, not ours — and you shouldn't be locked into one distribution from day one. GLIA boots straight to the terminal with the assistant already onboard: that's not an empty shell, it's already your personal textual desktop — a complete, working environment shaped around what you actually need, nothing you didn't choose. From there you grow it at your own pace: want a graphical desktop, a browser, a dev stack on top? You ask glia, and you install it together — because you decided you wanted it, not because it came bundled. And since the assistant runs on Arch, Debian and Fedora, you're just as free to bring it to the distro you already prefer. Either way you start bound to nothing, and you understand your computer because you built it yourself.

Way 1

Full install on a real PC

Build the ISO, write it to a USB stick and install with the Calamares graphical installer (UEFI or BIOS).

# build the ISO on an Arch host
sudo pacman -S archiso
sudo bash scripts/glia-build.sh
Way 2

Try it in a VM

Build the ISO and boot it in QEMU (UEFI, ~12 GB RAM for the embedded 7B model). No hardware touched.

# after building out/glia-*.iso
qemu-system-x86_64 -enable-kvm \
  -m 12G -cdrom out/glia-*.iso
Way 3 · easiest

Only the assistant

Add glia to your current Linux. One script for Arch, Debian & Fedora. Preview everything first with --dry-run.

git clone https://github.com/micheleperani76/GLIA glia
cd glia
bash scripts/install-assistant.sh --dry-run
bash scripts/install-assistant.sh
What's next

Roadmap & help wanted

GLIA is actively evolving. Contributions are welcome — open an issue or a pull request.

  • Cross-distro testing — help wanted.The assistant runs on Arch, Debian and Fedora, but the automated installer is tested on Arch/CachyOS only. Debian and Fedora need testers.
  • macOS support — help wanted (contributor-led).The assistant only, never a standalone distro — pure GLIA support for Mac users. Needs a Homebrew path, ollama serve/launchd instead of systemd, and a modern bash. Looking for a contributor to own and maintain it.
  • GNOME Shell search provider.Call glia straight from the GNOME overview — type a request, get the command without opening a terminal first.
  • Btrfs snapshots as a safety net.Plus branding and polish (phase 5).
  • Lightweight memory retrieval.If the stored facts outgrow the prompt.
  • A terminal-first desktop — the long-term vision.The most ambitious idea: a minimal desktop where the terminal is the backdrop, in place of the wallpaper. GUI programs open as windows you can stash and reopen, on a lightweight window manager underneath. The shell + AI become how you run the machine — launch, hide, reopen and arrange programs, and interact with the graphics, all by talking to glia. A desktop grown outward from the command line, not bolted on top of it.
