Command reference

Every glia command, explained

What each group is, why it exists, and what it does. The command is glia (you can rename the assistant — say to mypc — but glia always stays as a fallback). Flags follow standard conventions: -h, -V, -a, -l.

Green by design — commands that skip the AI

Every question to the model costs GPU time, energy and, on some setups, money. GLIA is built so you don't pay that cost twice. Turn any command you repeat into an alias that runs instantly with no AI call (glia -a add), and repeated questions are served from a local answer cache instead of re-running the model. The AI is there while you explore; it steps aside the moment you already know what you want. Fewer model calls means less energy — and a faster terminal.

Start here

What it is: the two things you always have, before any request. Why it matters: the built-in help is never more than one command away, and the assistant is yours to name. Give it a name that feels right and the terminal becomes your own textual desktop — a command line that answers to you, so working with your machine feels personal instead of intimidating.

glia -h · --helpThe full built-in help, always reachable from the terminal — offline, and even if you forget everything else on this page.
glia <area> helpA detailed help per area, each one listing every subcommand it has: -a help (shortcuts) · -m help (the AIs) · -p help (edit files) · --new help (new projects) · -T help (translate) · -w help (web search) · -i help (interactive) · --memory help · --update help. This page mirrors them; the terminal always has the last word.
glia -V · --versionWhich version you're running — and whether a newer one exists: online it runs the same check as --update --check (the two can never disagree); offline it shows the last cached answer, dated, in the past tense.
glia --rename <name>Rename the assistant to whatever feels right — e.g. mypc — to build a real rapport with your machine. The glia command always stays as a fallback, so you can never lock yourself out.
A tool you name is a tool you trust. Renaming the assistant is a small thing that changes the feel of everything: it stops being the terminal and becomes yours — your textual desktop, answering to the name you chose. And because glia never goes away, you can always find your way back.
01

The basics

What it is: you type a request in plain language and GLIA turns it into a real shell command. Why it exists: you should always see the command before it runs — you read it, approve it, and little by little you learn it. The AI proposes, you decide.

glia <request>Proposes a command. Enter runs it, n cancels, r retries, m edits it in place before running, e asks the AI to explain it — or type a hint (any text) right at the prompt to steer the AI toward a better command (hints accumulate).
gliaOn its own, opens the interactive prompt (see Interactive mode).
cat file | glia <request>Piped input becomes context for the AI — e.g. cat error.log | glia why does it fail.
rm -rf · dd · mkfs · curl|shDestructive commands are explained by the AI first, then ask a short s / y / j confirmation (Enter = no). Routine sudo commands don't trip this — only genuinely destructive patterns do.
(root needed)Commands that need root get sudo added automatically.
(on failure)Enter = ask for a fix, or type a hint to guide the AI (hints accumulate, up to 5 tries).
Every executed command is logged to ~/.local/share/glia/glia.log, together with the request that produced it.
02

Ask & explain

What it is: a plain-language answer with nothing executed — from the model alone, or from the live web when the answer is newer than the model knows. Why it exists: sometimes you want to understand a tool, not run it; and sometimes you need a current fact — a version, a release date, some recent news — that no offline model can have.

glia --ask <question>Explains in words, runs nothing. Follow-ups work — it remembers this terminal's recent conversation. It knows GLIA's own commands too (same sheet as the chat, same /contesto switch).
glia -d <question>Legacy short form of --ask — still works.
glia -cOne question and it's over? Then you want a chat instead — see Chat mode (new in v2.20).
glia -w <question>Search the web and answer, with sources. Uses the w3m text browser to query the chosen engine (DuckDuckGo by default) — no API key — then the model summarizes the results and ends with a Sources list. Ideal for anything past the model's training: latest versions, release dates, recent facts.
glia -w+ <question>Deep mode: also reads the top result pages in full for more detail (slower). Plain -w reads only the result snippets (faster).
glia -ws <search | URL>Direct results, no AI (new in v2.18): titles, URLs and snippets straight from the engine, in a couple of seconds — for quick lookups. Give it a URL or bare domain (glia -ws www.morrolinux.it) and it opens that page directly, paged.
glia -w bing: <question>Engine prefix (new in v2.18): ddg: bing: searx: pick the engine for this one search. Works with -w, -w+ and -ws.
glia --web-engine [engine]Show or switch the default search engine: ddg | bing | searx. For SearXNG set your instance first: glia --web-engine searx <URL> — searx refuses to become the default without one.
glia -w helpDetailed help for web search, with the tunable settings.
glia -m <name> -w <question>Run a web search on a chosen AI just for this call. The RAM is swapped only if needed, and your default AI is restored afterwards.
glia --web-model <name>Pin a dedicated AI for web search. --web-model default (or -d) follows your default; --web-model alone shows it and offers a guided menu.
Examples: glia --ask what does rsync do (offline explanation) · glia -w latest stable linux kernel (web + sources) · follow-ups work in --ask (it remembers the recent conversation of this terminal) · piped input is understood here too: cat script.sh | glia --ask what does this do

Web search needs w3m — the setup script installs it, and glia --doctor checks it. It runs entirely on your machine: w3m fetches, the local model summarizes. No account, no key.

Tip: w3m is also a text browser you can drive by hand — w3m example.com (Enter follows a link, B back, U open a URL, q quit).
03

Chat mode v2.20 → v3.2: dice, web, source, compact

What it is: a real conversation with the model — the whole dialogue is sent back at every turn, so and on Debian? means what it should. Why it exists: --ask answers once and exits; the difference is between asking a stranger and talking to someone who is actually listening. What is different here: the model knows what GLIA is — the command sheet goes to it, in -c and in --ask alike, so it can explain glia -T instead of inventing a flag (one switch for both: /contesto help off) — and a bar under every answer shows how full the window is, from the real token counts Ollama returns.

glia -cOpens the chat (long form --chat). Type at the prompt; up/down arrows recall earlier input.
glia -c <question>Opens the chat and asks the first question right away.
/esciQuit the chat — also /exit, /quit or Ctrl+D.
/nuovaStart over: the conversation is cleared. The AI forgets the dialogue, not your stored facts.
/salvaSave the whole conversation to a timestamped .md file in ~/.local/share/glia/chats.
/modello [name|n]Switch AI on the fly — this chat only: your default (-m) is untouched. Bare /modello lists the AIs and asks. The old model is unloaded first, so two never sit in RAM together — and the window is re-probed, because it belonged to the old AI.
/contestoWhat the AI is carrying, and what it costs (new in v2.21): each block with its real token count, asked of the engine — not chars/4.
/contesto help offTurn a block off for good: the sheet stays out of this chat and the next ones. On a small machine you do it once and forget it. Blocks: help (the command sheet), memory (your facts).
/ricorda <fact>Store a fact without leaving the chat (new in v2.21) — same memory --remember uses, so it is there tomorrow in every mode. The system message is rebuilt on the spot: the model sees it now, not next time.
/dadi 2d6Green tools in the chat (new in v2.24): a dice roll inside the sentence — lancio una palla di fuoco che fa /dadi 8d6 — resolves on the spot (no AI, shuf, zero tokens) and the model receives the sentence with the result already in it: you roll, the AI narrates. The roll is echoed in green before the model speaks. Aliases /wuerfel /roll; same notation as glia -D.
/web <question>The net INTO the conversation (new in v3.1): grab a piece of the web without leaving the chat. The collection is the -ws pipeline — w3m asks the engine, no AI, no model swap, the dialogue stays exactly where it is. The results (titles, URLs, snippets) join the conversation as a marked block with [n] sources, and the model uses them from your next message — like the dice: the tool brings a fact, the model narrates. Refused in source mode, where the document is the only truth. Aliases /cerca /suche.
/fonte <file>Source mode (new in v2.25): ONE document becomes the chat's only knowledge base — for studying, or an RPG played by your house rules. The whole file goes to the model with the anti-invention rule (what is not in the document does not exist: say so), and the AI is told to cite the part it draws from. Sheet and memory are suspended, not turned off — fewer distractions, fewer hallucinations, more room; /fonte off brings them back. The size is checked against the window before loading. At launch: glia -c --fonte <file> [first question]. Aliases /source /quelle. For documents larger than the window, see RAG in the ROADMAP.
/compattaShrink without starting over (new in v3.2): the middle way between the red bar and /nuova. The full conversation is saved to a .md first (safety net before scissors), then the model summarizes it and the dialogue becomes: base (untouched) + the summary marked as a summary + the last 2 exchanges verbatim — the immediate thread stays exact. A summary is lossy by definition and costs one inference: both declared, never hidden. Aliases /compact /kompakt.
/memoria · /scorda <n>List the stored facts · drop fact n. Same memory as --memory and --forget.
/aiutoThe command reminder, inside the chat.
glia -c helpThe full help page for chat mode.
The bar: [████░░░░░░░░░░░░░░░░] 21% · 6.9k/32.7k tok · turn 3 · qwen2.5-coder:7b
Green = relaxed · yellow = past half · red = the model is about to forget the beginning — /compatta to keep going, or /salva then /nuova. It does not start at 0%, and that is deliberate: the base (command sheet, facts) takes room, and a bar that hid it to look tidy would just be a bar that lies. /contesto shows the cost measured and switches it off.

The window tunes itself (new in v2.21): CHAT_NUM_CTX="auto" asks the model how much it can take (/api/show) and fits it to the free RAM — two limits, the smaller wins, and the startup line always says which. v2.20 shipped 8192 by hand; the machine it was written on takes 32768. Put a number back in and it wins. glia --doctor reports the window too.

The AI knows the commands, and that sheet is not a hand-written copy: it is show_help, the same function glia -h prints — so it cannot drift into reciting flags that no longer exist.

Commands are localized: /esci /nuova /salva /modello /contesto /ricorda /memoria /scorda /dadi /web /fonte /compatta /aiuto (it) · /ende /neu /speichern /modell /kontext /merken /gedaechtnis /vergiss /wuerfel /suche /quelle /kompakt /hilfe (de) · /exit /new /save /model /context /remember /memory /forget /roll /web /source /compact /help (en) — and every alias works in every language. Anything that is not a /command is conversation.

Which mode when: glia <phrase> proposes and runs commands · glia --ask gives one answer and exits · glia -c is a discussion that keeps its context.
04

Shortcuts no AI call

What it is: a shortcut saves a command you use often, so you recall it by name without ever asking the AI. Why it exists: the fastest, greenest path — an alias runs instantly and spends zero GPU. Ask the model once, save the result, reuse it forever.

glia -aList your shortcuts (numbered).
glia -a <name|n>Run a shortcut, by name or number.
glia -a addGuided create — asks for name, command and type.
glia -a add <name> <cmd>Create a direct shortcut on the fly.
glia -a rmGuided remove — shows the list, asks the number.
glia -a rm <n|name>Remove by number or name (with confirmation).
glia -a saveSave the last successful command as a shortcut.
glia -a editOpen the shortcuts file in your editor.
Two types. Direct — runs the saved command immediately. With-AI — shows the command and asks: Enter = use the shortcut, a = ask the AI instead (handy when the original request was vague).

Names are case-insensitive · removal always confirms (Enter = no) · if you repeat the same request within 10 minutes, GLIA offers once to save it as a shortcut · file: ~/.config/glia/aliases
05

Green tools no AI, ever — and open to new ones

What it is: a toolbox of small, deterministic commands that never touch the model — instant, offline, zero tokens. Why it exists: a language model gets arithmetic and randomness wrong; these tools never do. The model narrates, the tools bring the facts. The point to steal: every tool works standalone at the prompt AND inside the integrations — in the chat it resolves in the middle of your sentence (/calc, /conv, /dadi...), and the registry is open: a new tool is one function and one line, not a redesign.

glia -D 2d6RPG dice (long form --dice): 1d4, 2d6, d20, 1d100+4, several rolls per call, single dice shown (2d6 → 4 + 5 = 9) — from shuf, the kernel's entropy. In chat: /dadi, inside the sentence. glia -D help for details.
glia -R 100Random number (long form --random): the institutional cousin of the dice — -R 100 draws 1–100, -R 5-50 in a range. Same kernel entropy, none of the RPG flavour. In chat: /caso.
glia -X "340*1.22"Calculator (long form --calc): awk arithmetic — = 414.8. Decimal comma accepted, ^ is power. Letters refused outright: a calculator that only knows numbers cannot be talked into anything. In chat: /calc — and the model stops doing sums it would get wrong.
glia --conv 100 mi kmUnit converter: lengths, masses, temperatures (°C/°F/K), volumes, areas, speeds, data (GB vs GiB, finally settled by arithmetic), energy, pressure, time — 100 mi = 160.934 km. Different categories are refused: apples do not convert into kilometres. Full unit list: glia --conv help. In chat: /conv 100 mi km.
glia --days 2026-12-25Day counter: how many days until (or since), and the weekday — 2026-12-25 (venerdì) → tra 159 giorni. Two dates = the span between them. DD/MM/YYYY accepted too. In chat: /giorni.
glia --pw 16Password / UUID generator: straight from /dev/urandom (8–128 chars, default 20; --pw uuid for a UUID). Deliberately absent from the chat: a password that transits a model's context is no longer a secret.
glia --pick a b cDraw from a list: who does the dishes — → sushi; --pick -n 2 draws two. In chat: /scegli a b c, and the result drops into the dialogue.
glia --toolsThe whole toolbox on one page, with the chat aliases and the unit list.
Why "green": every one of these answers in milliseconds with Ollama stopped, on the smallest server, spending zero GPU. And in the chat they are not commands beside the conversation — they resolve inside it: il forziere dista /conv 30 yd m e per scassinarlo tiro /dadi 1d20+5 reaches the model with both facts already computed. The tool brings the fact, the AI narrates.

Memory

What it is: GLIA can remember short facts — machines, paths, habits — and adds them to the context of each request. Why it exists: so the AI keeps your setup in mind without you repeating it every time.

glia --remember "<fact>"Remember a fact.
glia --memoryList the remembered facts (numbered).
glia --forget <n>Forget fact number n.
Max 20 facts (the oldest are dropped) · a fact is used only when the request concerns it · file: ~/.config/glia/memory
07

Models / AI

What it is: the models are downloaded and kept by Ollama; GLIA uses one at a time. Why it exists: different machines and tasks want different models — pick the right one, or override it just for a single command.

glia -mList downloaded models (numbered) and choose the default (guided). glia -m list and glia -m ls do the same. Each model shows the roles pinned onto it — default · web · project · translate — so you see at a glance which AI serves which command, e.g. 1) gemma4:latest ← default · web. One AI can hold several roles, or none.
glia -m <n|name>Set the default model.
glia -m <n|name> <task>Use that model just for this command — default unchanged.
glia -m pullGuided download: checks RAM, GPU and disk, then offers up to 10 models that actually FIT this machine — pick one by number. Ends by showing the manual command, so you can do it yourself.
glia -m pull <name>Download a model by name (free-RAM check first, then offers to make it the default).
glia -m update [n|name]Re-pull every downloaded model — or just that one. Ollama only fetches versions that changed.
glia -m rm <n|name>Remove a model (asks confirmation; clears the saved default if it was the one removed).
glia -m psShow which model is loaded in RAM right now (same as ollama ps).
glia -m stop <n|name>Unload a model from RAM immediately (same as ollama stop) — frees memory without deleting the download.
glia -m roleOne console for the AI roles. Lists your downloaded AIs, numbered, each tagged with the job it holds — default · web · project · translate — exactly as glia -m numbers them. Assign by number: glia -m role <n> <role> (role is the full name or its initial: web/w, project/p, translate/t); glia -m role 0 <role> sends that job back to the default. The single place to SEE and MOVE the roles that --web-model / --project-model / --translate-model still set one at a time.
glia -m benchMeasures CPU vs Intel iGPU generation speed on THIS machine and prints a verdict — instead of asking you to edit a systemd override and read two numbers yourself. Needs sudo (restarts ollama.service twice), asks for confirmation first, and always restores the original state afterwards, even if interrupted. Intel iGPU only for now; a dedicated NVIDIA/AMD GPU is refused with a clear message. glia -m bench --dry-run shows exactly what it would do without changing anything.
glia --updateUpdate GLIA itself (the program), to the newest git tag on your channel. The running version is backed up first, the new one is validated before it replaces anything, and the swap is atomic. Alias: -U. glia --update help is the full guide.
glia --update --checkAsk whether a newer version exists on your channel and install nothing.
glia --channel [stable|beta]Show or switch the release channel. stable (default) sees only finished versions (vX.Y.Z); beta also sees previews (-beta.N, -rc.N).
glia --rollbackPut a previously installed version back, from the automatic backups in ~/.local/share/glia/versions (last 3 kept). The rollback backs up the current one too, so it is itself reversible.
glia --update-engineUpdate the Ollama engine itself, using this machine's package manager. (This was bare glia --update before v2.17.)
Same words as Ollama, on purpose: ollama list · ollama pull <name> · ollama rm <name> · ollama ps · ollama stop — what you learn here works there too.

Switching the default model unloads the previous one from RAM automatically, so you never keep two models resident at once. When a command needs a different AI (-w, -p, or a one-off -m), the default is swapped out first and brought back afterwards — but if you stopped the default yourself (ollama stop), GLIA respects that and leaves it off.

Default saved in ~/.config/glia/model · the feasible-model list comes from glia-hardware -l (curated catalog + this machine's hardware).
08

Code: edit files (-p) and new projects (--new)

What it is: two opposite jobs, two flags. -p changes a file you already have; --new plans a new project and writes its files. Both confirm at every step. Why it exists: for work that goes beyond a single command — still transparent, still one approval at a time. Heads-up (v2.18): -p used to be the generator. It is now the editor, and the generator moved to --new. glia -p something-that-is-not-a-file tells you so instead of guessing — a typo in a filename must never silently scaffold a project.

glia -p <file> "<request>"Edit a file that already exists: shows the diff and the exact git apply command, then asks. New in v2.18.
glia -p <f1> <f2> "<request>"The same on several files at once: one diff, one confirmation, one commit. New in v2.18.
glia -p --undoUndo the last -p commit. It uses git revert, never git reset: the undo is itself a commit, so your history is never rewritten.
glia --new <idea> · -nPlan the steps and write the files of a new project (you confirm each one). This was -p up to v2.17.
glia --project-model <name>Pin a dedicated AI for project mode (the twin of --web-model). --project-model default (or -d) follows your default; --project-model alone shows it and offers a guided menu.
-p: at the diff promptthe confirm key follows your language (en y · it s · de j) = apply, Enter or n = cancel, e = rewrite the request, q = quit. Cancel and the patch stays in /tmp: apply it by hand whenever you like.
--new: at the plan promptEnter = proceed, n = cancel, or type a hint — the plan is revised with it, without losing what the original request asked for. Hints accumulate.
--new: at each fileEnter = save, v = view all, r = regenerate, type a hint = regenerate following it, s = skip.
-p never lets the AI write the diff: small models get line numbers wrong, so the patch would never apply. The AI sends search/replace blocks, GLIA applies them to a temporary copy and computes the real diff itself — so what you see always applies cleanly. Blocks that don't match are retried with the error explained, then GLIA asks for the whole file rewritten, then it gives up honestly and shows you the raw reply. Your file is never left half-done. It needs a git repo (that's the undo story) and proposes git init when there isn't one. A file too big for the model's context is refused, not truncated — a mutilated file produces wrong edits. Once the change is applied, GLIA offers an optional local commit (never pushed): that commit is what makes glia -p --undo possible. Log: ~/.config/glia/pmode.log. Requires git and jq.

--new: list the files you want explicitly, so the AI doesn't forget them · where the project lands: once you have renamed the assistant it gets its own folder, and projects are created in ~/<name>/projects/<project-name> — e.g. ~/arx/projects/backup-script. While the assistant is still called glia there is no dedicated folder: the project is created where you are ($PWD), leaving the ~/glia repo untouched. If the folder can't be written, it falls back to $PWD (or to your home for the glia anchor) · it never touches an existing folder: if the name is taken, a free one is picked (name-2, name-3, …) and shown before writing · requires jq.

Examples: glia -p backup.sh "add a --verbose flag" · glia --new a backup script with rsync and a README explaining it
09

Translate a file

What it is: you hand it a file, it writes the translation into a new file next to it. Why it exists: a README, a script's comments, some notes — translating them by hand is tedious, and pasting them into a web service means shipping your work to someone else's computer. This stays on your machine. The original is never touched: the translation is always a new file, so there is nothing to undo.

glia -T <file>Translate into the interface language (the one from glia --lang).
glia -T <file> <lang>Translate into that language — it, en, de, or the full name: anything the three codes don't cover is handed to the AI as you wrote it, so glia -T notes.md portuguese works and lands in notes.portuguese.md.
glia -T helpThe detailed help for this area. glia --translate-model help lands here too.
glia --translate-modelShow the AI currently pinned for translations and open the guided menu.
glia --translate-model <name>Pin a dedicated AI for translations (the same idea as --web-model and --project-model).
glia --translate-model -dGo back to following the default AI. With nothing pinned there is no model swap and no waiting.
glia -m <name> -T <file>Use that AI only for this translation; your default is restored afterwards.
(while translating)The text appears as the AI writes it — no invented percentage, you watch the work. At the end: Enter = save · r = redo · type a hint = redo following it · s = discard.
Where it lands. README.md → README.en.md: the name is shown before anything is written, and if it already exists a free one is picked (.en-2, …). The original file is never modified.

It checks its own work (the same checks as --new): a translated .md must still be Markdown, a .sh must pass bash -n. If a check fails it retries with the error explained, then saves with a warning rather than pretending. In code only comments and messages are translated — never code, commands, variables or URLs. Binary files, and files too big for the model's context, are refused, not truncated.

Examples: glia -T README.md · glia -T README.md en · glia -T backup.sh de (comments and messages only, code untouched)

Interactive mode

What it is: a dedicated prompt where special characters are taken literally. Why it exists: the shell interprets ' " $ | ; & * ( ) ` before GLIA ever sees them, breaking a sentence that contains them. The interactive prompt reads your request character by character instead.

gliaOn its own, enters the prompt glia> — a real REPL: type request after request, the conversation context carries over.
glia -iSame, explicit form. Empty Enter = exit.
Use it whenever the request contains ' " $ | ; & *. Type at the glia> prompt, e.g. count the *.log files in the current folder or how much space does $HOME use. Simple sentences with no symbols still work inline: glia find the largest files in /home.

12

Settings & identity

What it is: naming, language and housekeeping. Why it exists: the assistant is yours to name and tune — and you can always find it again, because glia never goes away.

glia --rename <name>Give the assistant a new name (e.g. mypc). The glia command always stays as a fallback. A nickname is a command in your PATH, so since v2.18.2 a name that already exists is refused, not silently buried: reserved names (ls, sudo, rm, git, glia, ollama…) never pass, and any other existing command is refused too — showing you the real path it would have covered, plus free names to use instead. One rename, one name: the previous nickname is removed, and the removal is announced with the command that did it.
glia --lang <it|en|de>Change the interface language.
glia --doctorOne-shot health check, grouped in clear sections (v2.18): tools · engine and model · GPU and backend · command and folders · version and updates — with a fix hint for anything wrong. Checks that pass stay visible; the model check speaks only when the model is missing. Since v2.18.3 it also catches a nickname that shadows a real command — the ones made before the rename guard existed — and tells you exactly what it covers.
glia --dangerThe rules that decide which commands get the warning, the AI's explanation and the confirm prompt — listed, numbered, built-in first and yours after. The built-ins are ours and cover the classics (rm -rf, dd, mkfs, fork bombs, curl|sh); the real dangers are also yours, so you add them: glia --danger add 'terraform .*destroy'. Yours are added to the built-ins, never instead of them, and --danger rm <n> only removes your own — the built-ins don't come off, like --rename's reserved names: the cost is a keypress, the risk they cover is your disk. glia --danger test '<cmd>' is the dry-run of safety: it says whether a rule would fire and which one, running nothing. A rule never blocks — it asks once more, with the AI explaining the command first (so the explanation works for your rules too). File: ~/.config/glia/danger, plain text, one regex per line.
glia --clear-cacheEmpty the answer cache. The old spelling --cache-clear still works.
glia --kaboomGuided uninstall, two levels: 1 = only the program (engine and AIs stay), 2 = everything. Every command is shown first, then it asks you to type the confirm word in your language (en YES · it SI · de JA) — Enter alone cancels. Shared tools (curl, jq) are never touched.
glia -V · --versionShow the version, with a real update check when online (offline: the last cached answer, dated).
glia -hShow the built-in help.
Recovery. Even after you rename the assistant to, say, mypc, the glia command stays active as an emergency passkey — so you can never lock yourself out of your own assistant.
13

Hardware helper

What it is: a companion command that reads your machine and recommends the right model tier. Why it exists: picking a model that's too big stalls; too small underperforms. Let the hardware decide.
