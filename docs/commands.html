Command reference

Every glia command, explained

What each group is, why it exists, and what it does. The command is glia (you can rename the assistant — say to mypc — but glia always stays as a fallback). Flags follow standard conventions: -h, -V, -a, -l.

Green by design — commands that skip the AI

Every question to the model costs GPU time, energy and, on some setups, money. GLIA is built so you don't pay that cost twice. Turn any command you repeat into an alias that runs instantly with no AI call (glia -a add), and repeated questions are served from a local answer cache instead of re-running the model. The AI is there while you explore; it steps aside the moment you already know what you want. Fewer model calls means less energy — and a faster terminal.

Start here

What it is: the two things you always have, before any request. Why it matters: the built-in help is never more than one command away, and the assistant is yours to name. Give it a name that feels right and the terminal becomes your own textual desktop — a command line that answers to you, so working with your machine feels personal instead of intimidating.

glia -h · --helpThe full built-in help, always reachable from the terminal — offline, and even if you forget everything else on this page.
glia -V · --versionWhich version you're running.
glia --rename <name>Rename the assistant to whatever feels right — e.g. mypc — to build a real rapport with your machine. The glia command always stays as a fallback, so you can never lock yourself out.
A tool you name is a tool you trust. Renaming the assistant is a small thing that changes the feel of everything: it stops being the terminal and becomes yours — your textual desktop, answering to the name you chose. And because glia never goes away, you can always find your way back.
01

The basics

What it is: you type a request in plain language and GLIA turns it into a real shell command. Why it exists: you should always see the command before it runs — you read it, approve it, and little by little you learn it. The AI proposes, you decide.

glia <request>Proposes a command. Enter runs it, n cancels, r retries, m edits it in place before running, e asks the AI to explain it — or type a hint (any text) right at the prompt to steer the AI toward a better command (hints accumulate).
gliaOn its own, opens the interactive prompt (see Interactive mode).
cat file | glia <request>Piped input becomes context for the AI — e.g. cat error.log | glia why does it fail.
rm -rf · dd · mkfs · curl|shDestructive commands are explained by the AI first, then ask a short s / y / j confirmation (Enter = no). Routine sudo commands don't trip this — only genuinely destructive patterns do.
(root needed)Commands that need root get sudo added automatically.
(on failure)Enter = ask for a fix, or type a hint to guide the AI (hints accumulate, up to 5 tries).
Every executed command is logged to ~/.local/share/glia/glia.log, together with the request that produced it.
02

Ask & explain

What it is: a plain-language answer with nothing executed — from the model alone, or from the live web when the answer is newer than the model knows. Why it exists: sometimes you want to understand a tool, not run it; and sometimes you need a current fact — a version, a release date, some recent news — that no offline model can have.

glia --ask <question>Explains in words, runs nothing. Follow-ups work — it remembers this terminal's recent conversation.
glia -d <question>Legacy short form of --ask — still works.
glia -w <question>Search the web and answer, with sources. Uses the w3m text browser to query DuckDuckGo — no API key — then the model summarizes the results and ends with a Sources list. Ideal for anything past the model's training: latest versions, release dates, recent facts.
glia -w+ <question>Deep mode: also reads the top result pages in full for more detail (slower). Plain -w reads only the result snippets (faster).
glia -w helpDetailed help for web search, with the tunable settings.
glia -m <name> -w <question>Run a web search on a chosen AI just for this call. The RAM is swapped only if needed, and your default AI is restored afterwards.
glia --web-model <name>Pin a dedicated AI for web search. --web-model default (or -d) follows your default; --web-model alone shows it and offers a guided menu.
Examples: glia --ask what does rsync do (offline explanation) · glia -w latest stable linux kernel (web + sources) · follow-ups work in --ask (it remembers the recent conversation of this terminal) · piped input is understood here too: cat script.sh | glia --ask what does this do

Web search needs w3m — the setup script installs it, and glia --doctor checks it. It runs entirely on your machine: w3m fetches, the local model summarizes. No account, no key.

Tip: w3m is also a text browser you can drive by hand — w3m example.com (Enter follows a link, B back, U open a URL, q quit).
03

Shortcuts no AI call

What it is: a shortcut saves a command you use often, so you recall it by name without ever asking the AI. Why it exists: the fastest, greenest path — an alias runs instantly and spends zero GPU. Ask the model once, save the result, reuse it forever.

glia -aList your shortcuts (numbered).
glia -a <name|n>Run a shortcut, by name or number.
glia -a addGuided create — asks for name, command and type.
glia -a add <name> <cmd>Create a direct shortcut on the fly.
glia -a rmGuided remove — shows the list, asks the number.
glia -a rm <n|name>Remove by number or name (with confirmation).
glia -a saveSave the last successful command as a shortcut.
glia -a editOpen the shortcuts file in your editor.
Two types. Direct — runs the saved command immediately. With-AI — shows the command and asks: Enter = use the shortcut, a = ask the AI instead (handy when the original request was vague).

Names are case-insensitive · removal always confirms (Enter = no) · if you repeat the same request within 10 minutes, GLIA offers once to save it as a shortcut · file: ~/.config/glia/aliases
04

Memory

What it is: GLIA can remember short facts — machines, paths, habits — and adds them to the context of each request. Why it exists: so the AI keeps your setup in mind without you repeating it every time.

glia --remember "<fact>"Remember a fact.
glia --memoryList the remembered facts (numbered).
glia --forget <n>Forget fact number n.
Max 20 facts (the oldest are dropped) · a fact is used only when the request concerns it · file: ~/.config/glia/memory
05

Models / AI

What it is: the models are downloaded and kept by Ollama; GLIA uses one at a time. Why it exists: different machines and tasks want different models — pick the right one, or override it just for a single command.

glia -mList downloaded models (numbered) and choose the default (guided). glia -m list and glia -m ls do the same. Each model shows the roles pinned onto it — default · web · project — so you see at a glance which AI serves which command.
glia -m <n|name>Set the default model.
glia -m <n|name> <task>Use that model just for this command — default unchanged.
glia -m pullGuided download: checks RAM, GPU and disk, then offers up to 10 models that actually FIT this machine — pick one by number. Ends by showing the manual command, so you can do it yourself.
glia -m pull <name>Download a model by name (free-RAM check first, then offers to make it the default).
glia -m update [n|name]Re-pull every downloaded model — or just that one. Ollama only fetches versions that changed.
glia -m rm <n|name>Remove a model (asks confirmation; clears the saved default if it was the one removed).
glia -m psShow which model is loaded in RAM right now (same as ollama ps).
glia -m stop <n|name>Unload a model from RAM immediately (same as ollama stop) — frees memory without deleting the download.
glia --updateUpdate GLIA itself (the program), to the newest git tag on your channel. The running version is backed up first, the new one is validated before it replaces anything, and the swap is atomic. Alias: -U. glia --update help is the full guide.
glia --update --checkAsk whether a newer version exists on your channel and install nothing.
glia --channel [stable|beta]Show or switch the release channel. stable (default) sees only finished versions (vX.Y.Z); beta also sees previews (-beta.N, -rc.N).
glia --rollbackPut a previously installed version back, from the automatic backups in ~/.local/share/glia/versions (last 3 kept). The rollback backs up the current one too, so it is itself reversible.
glia --update-engineUpdate the Ollama engine itself, using this machine's package manager. (This was bare glia --update before v2.17.)
Same words as Ollama, on purpose: ollama list · ollama pull <name> · ollama rm <name> · ollama ps · ollama stop — what you learn here works there too.

Switching the default model unloads the previous one from RAM automatically, so you never keep two models resident at once. When a command needs a different AI (-w, -p, or a one-off -m), the default is swapped out first and brought back afterwards — but if you stopped the default yourself (ollama stop), GLIA respects that and leaves it off.

Default saved in ~/.config/glia/model · the feasible-model list comes from glia-hardware -l (curated catalog + this machine's hardware).
06

Code: edit files (-p) and new projects (--new)

What it is: two opposite jobs, two flags. -p changes a file you already have; --new plans a new project and writes its files. Both confirm at every step. Why it exists: for work that goes beyond a single command — still transparent, still one approval at a time. Heads-up (v2.18): -p used to be the generator. It is now the editor, and the generator moved to --new. glia -p something-that-is-not-a-file tells you so instead of guessing — a typo in a filename must never silently scaffold a project.

glia -p <file> "<request>"Edit a file that already exists: shows the diff and the exact git apply command, then asks. New in v2.18.
glia --new <idea>Plan the steps and write the files of a new project (you confirm each one). This was -p up to v2.17.
glia --project-model <name>Pin a dedicated AI for project mode (the twin of --web-model). --project-model default (or -d) follows your default; --project-model alone shows it and offers a guided menu.
-p: at the diff prompts/y = apply, Enter or n = cancel, e = rewrite the request, q = quit. Cancel and the patch stays in /tmp: apply it by hand whenever you like.
--new: at the plan promptEnter = proceed, n = cancel, or type a hint — the plan is revised with it, without losing what the original request asked for. Hints accumulate.
--new: at each fileEnter = save, v = view all, r = regenerate, type a hint = regenerate following it, s = skip.
-p never lets the AI write the diff: small models get line numbers wrong, so the patch would never apply. The AI sends search/replace blocks, GLIA applies them to a temporary copy and computes the real diff itself — so what you see always applies cleanly. Blocks that don't match are retried with the error explained, then GLIA asks for the whole file rewritten, then it gives up honestly and shows you the raw reply. Your file is never left half-done. It needs a git repo (that's the undo story) and proposes git init when there isn't one. A file too big for the model's context is refused, not truncated — a mutilated file produces wrong edits. Log: ~/.config/glia/pmode.log. Requires git and jq.

--new: list the files you want explicitly, so the AI doesn't forget them · the project is created where you are — <current folder>/<project-name> — so you never lose your context (if the current folder is not writable, it falls back to ~/<name>-projects in your home) · it never touches an existing folder: if the name is taken, a free one is picked (name-2, name-3, …) and shown before writing · requires jq.

Examples: glia -p backup.sh "add a --verbose flag" · glia --new a backup script with rsync and a README explaining it
07

Interactive mode

What it is: a dedicated prompt where special characters are taken literally. Why it exists: the shell interprets ' " $ | ; & * ( ) ` before GLIA ever sees them, breaking a sentence that contains them. The interactive prompt reads your request character by character instead.

gliaOn its own, enters the prompt glia> — a real REPL: type request after request, the conversation context carries over.
glia -iSame, explicit form. Empty Enter = exit.
Use it whenever the request contains ' " $ | ; & *. Type at the glia> prompt, e.g. count the *.log files in the current folder or how much space does $HOME use. Simple sentences with no symbols still work inline: glia find the largest files in /home.
08

Log

What it is: the history of commands GLIA actually executed. Why it exists: transparency you can audit — see exactly what ran on your machine.

glia -lShow the log of executed commands.
glia --logSame, long form.
file: ~/.local/share/glia/glia.log
09

Settings & identity

What it is: naming, language and housekeeping. Why it exists: the assistant is yours to name and tune — and you can always find it again, because glia never goes away.

glia --rename <name>Give the assistant a new name (e.g. mypc). The glia command always stays as a fallback.
glia --lang <it|en|de>Change the interface language.
glia --doctorOne-shot health check: engine, model, RAM, dependencies, PATH, folders — with a fix hint for anything wrong.
glia --clear-cacheEmpty the answer cache.
glia --kaboomGuided uninstall, two levels: 1 = only the program (engine and AIs stay), 2 = everything. Every command is shown first, then it asks you to type YES to proceed. Shared tools (curl, jq) are never touched.
glia -V · --versionShow the version.
glia -hShow the built-in help.
Recovery. Even after you rename the assistant to, say, mypc, the glia command stays active as an emergency passkey — so you can never lock yourself out of your own assistant.
10

Hardware helper

What it is: a companion command that reads your machine and recommends the right model tier. Why it exists: picking a model that's too big stalls; too small underperforms. Let the hardware decide.

glia-hardwareHardware report (RAM / GPU / VRAM) and recommended models.
glia-hardware -jJSON output — designed for the installer.
Tiers range from a 3–4B model on RAM-only laptops up to 30B-class models on 20 GB+ GPUs. See the full table on the home page.
